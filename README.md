## 🚀 Setup Guide (Step by Step)

### 1. Install Flutter
Download Flutter SDK from https://flutter.dev/docs/get-started/install
Make sure Flutter Web is enabled:
```bash
flutter config --enable-web
flutter doctor
```

### 2. Install Firebase CLI & FlutterFire CLI
```bash
npm install -g firebase-tools
dart pub global activate flutterfire_cli
firebase login
```

### 3. Create Firebase Project
- Go to https://console.firebase.google.com
- Create a new project called `rizznight`
- Enable these services:
  - **Authentication** → Email/Password
  - **Firestore Database** → Start in production mode
  - **Storage** (for future photo uploads)

### 4. Connect Flutter to Firebase
Inside this project folder, run:
```bash
flutterfire configure
```
This auto-generates `lib/firebase_options.dart` with your real keys.

### 5. Deploy Firestore Rules & Indexes
```bash
firebase deploy --only firestore:rules
firebase deploy --only firestore:indexes
```

### 6. Install Dependencies
```bash
flutter pub get
```

### 7. Run the Web App
```bash
flutter run -d chrome
```

### 8. Build for Production
```bash
flutter build web --release
```
Output is in `build/web/` — deploy to Firebase Hosting:
```bash
firebase deploy --only hosting
```

## 🔐 Setting Up Admins

After creating admin user accounts in the app:

1. Go to Firebase Console → Firestore
2. Open the `users` collection
3. Find the admin user's document
4. Add a field: `isAdmin` = `true` (boolean)

Then update `lib/core/constants/app_constants.dart`:
```dart
static const List<String> adminUids = [
  'actual-firebase-uid-here',
  'second-admin-uid-here',
];
```

## 📸 Replacing Placeholder Photos

In `lib/features/landing/presentation/landing_page.dart`, find `_PhotosSection`:
```dart
static const List<Map<String, String>> _photos = [
  {'url': 'YOUR_DIRECT_IMAGE_URL', 'label': 'RUN #1'},
  ...
];
```

To get a direct Google Drive image URL:
1. Right-click image in Drive → Get link → Copy
2. Change the URL format from:
   `https://drive.google.com/file/d/FILE_ID/view`
   to:
   `https://drive.google.com/uc?export=view&id=FILE_ID`

## 🏗️ How the Slot System Works

1. Admin creates a run event (100 slots, open status)
2. Runners log in → go to Runs page → claim a slot
3. Run day happens
4. Admin goes to Admin Panel → Attendance tab
5. Admin sees full list of registered runners by name
6. Admin ticks ✓ for who showed up, ✗ for no-shows
7. Admin clicks "Confirm" → +3km instantly added to all ✓ runners
8. Action is irreversible (prevents double KM)
